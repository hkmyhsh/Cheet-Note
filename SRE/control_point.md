# 1. 「制御点を追加する」とは何か

**システムが“暴走・劣化・破綻”する前に、止める／減速させる仕組みを設けること**

## 制御点の工学的定義

**入力・処理・出力のどこかに設ける“振る舞いを制限・観測・調整できるポイント”**

```
入力 → 処理 → 出力
       ↑
    制御フィードバック
```

## 代表的な制御点の具体例
- タイムアウト
  - 処理が一定時間を超えたら止める。
    - → 暴走を防ぐ
- サーキットブレーカー
  - 一定回数失敗したら通信を遮断。
    - → 連鎖障害を止める
- レートリミット
  - 1秒あたりのリクエストを制限。
    - → 負荷集中を抑制
- キュー制限
  - 無限に溜めない。
    - → 遅延拡散を防ぐ
- エラーバジェット
  - SLO違反が一定を超えたらリリース停止。
    - → 品質劣化の加速を止める


## 制御点がない世界

- 自然成長に任せた状態

```
例：
	•	無制限リトライ
	•	タイムアウトなし
	•	接続数制限なし
	•	キュー無制限
	•	メモリ制限なし
```

- 障害は「発生する」ではなく「増幅される」構造

### 「思考の制御点がない」状態

	•	横入りタスクで疲労が爆増
	•	思考が止まらない
	•	レビュー後に頭が止まらない

# 制御点の種類
- 制限 → 上限を決める → タイムアウト
- 観測 → 状態を見える化 → メトリクス
- 反応 → 状態に応じて変える → オートスケール
