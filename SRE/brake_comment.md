# ブレーキ用コメント集

| 状態 | テンプレ |
| ------ | ------ |
| 軽い違和感 | ② |
| 前提が曖昧 | ① |
| 影響が広そう | ③ or ⑥ |
| 戻せない変更 | ④（必須） |
| 不安だが止めたくない | ⑧ |


## ① 前提確認（最も使用頻度高い）

```
この変更自体は問題なさそうです 👍  
一点だけ、前提を揃えたいです。

- 前提：〇〇（自分の理解）

この認識で合っていますか？
```

👉 効果：
	•	否定しない
	•	認識ズレだけを検知

## ② 軽い違和感の扱い（超重要）

```
大きな問題ではないのですが、少しだけ違和感があります。

- 気になった点：〇〇

自分の認識違いかもしれないので、一度だけ確認させてください。
```

👉 効果：
	•	「止めずに違和感を出せる」
	•	最重要テンプレ

## ③ 依存関係の切り出し

```
進め方は良いと思います 👍  
依存だけ軽く確認させてください。

- 影響しそうな範囲：〇〇

ここは今回の変更範囲に含まれていますか？
```

👉 効果：
	•	「止める」ではなく「範囲限定」

## ④ 不可逆変更の減速（重要）

```
一点だけリスク観点で確認させてください。

この変更、戻す場合の手順は想定されていますか？
（簡単で良いので把握したいです）
```

👉 効果：
	•	強ブレーキを自然にかける
	•	「安全側の人」として認識される

## ⑤ 認識ズレの補正

```
認識合わせだけさせてください。

成功条件は「〇〇になればOK」という理解で合っていますか？
```

👉 効果：
	•	後半の事故を潰す
	•	摩擦がほぼ出ない

## ⑥ 影響範囲の可視化

```
一点だけ影響範囲を見たいです。

- 影響しそうなユーザー / システム：〇〇

ここは想定通りでしょうか？
```

👉 効果：
	•	チーム全体の視界が広がる
	•	“止めた感”が出ない

## ⑦ 分割提案（止めない技術）

```
進め方として良さそうです 👍  

もし可能なら、
この部分だけ先に切り出して試すことはできますか？
```

👉 効果：
	•	スピードを維持しつつリスク低減

## ⑧ 時間制限付き減速

```
10分だけこの観点を見ても良いですか？

- 観点：〇〇

問題なければそのまま進めたいです。
```

👉 効果：
	•	長引くことへの不安を消す
	•	チームに受け入れられやすい
