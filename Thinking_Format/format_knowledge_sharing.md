# ナレッジ共有の設計

##課題

	•	自分の中では整理済み
	•	しかし他人には伝わらない
	•	「細かすぎる」と言われる
	•	もしくは「難しい」と言われる

 → **抽象度レイヤーの不一致問題**

## 原則
- **3層構造で出す**
  - 体験 → 具体 → 構造
- 直接「構造図」で出してはいけない
  - これをやりがち

## 本質的な役割

**「安定化装置」**

 → **不安定な組織の中で、安定を作る側の人間**

実行者でも管理者でもない


## 推奨フォーマット

### レベル1：現場視点（3行）

- 「何が困るのか」
- 「これをやると何が楽か」

### レベル2：実務テンプレ

- チェックリスト・手順・図

### レベル3：構造説明（必要な人だけ読む）

```
例（CI障害初動）

1️⃣ まずこれを見てください（5分で読める版）
2️⃣ 手順はこちら（Markdownテンプレ）
3️⃣ なぜこうなっているか（体系図）
```

## 「前提が違う人たち」との付き合い方

**全員を理解させようとしないこと。**

自分の役割は：

	•	反応を揃える
	•	判断を揃える
	•	手順を揃える

であって、

	•	思考を揃える

ではない
