# 業務再現性チェックリスト（個人用・軽量版）

> 目的：  
> ・日を跨いでも同じ品質で再開できる  
> ・他人に渡しても意図が崩れない  
> ・「考え直し」「やり直し」を減らす

---

## ① タスクに手をつける前（Start Check）
**誤着手・迷走防止（1分）**

- [ ] このタスク種別を言える（調査 / 設計 / 実装 / 修正）
- [ ] 今日のゴールを1文で書ける
- [ ] 今日「やらないこと」が1つ以上ある
- [ ] 成果物の保存場所が決まっている

→ **1つでも No があれば作業を始めない**

---

## ② 途中でその日を終える前（Stop Check）
**翌日の再起動コスト削減（3分）**

- [ ] 次にやる「1アクション」が書いてある
- [ ] 迷った／判断保留した点が残っている
- [ ] 未確定事項と保留理由が分かる
- [ ] 今日どこまでやったかが分かる

→ 「保存」ではなく**自分への引き渡し**

---

## ③ 日を跨いで再開するとき（Resume Check）
**状態復元を高速化（2分）**

- [ ] 前回のゴールを1分で読み返した
- [ ] 今日のゴールを再定義した
- [ ] 保留点に着手するか判断した
- [ ] 今やる作業は判断不要になっている

→ 読み返して迷ったら**作業に入らない**

---

## ④ 業務依頼として渡す前（Request Check）
**往復・誤解を防ぐ（5分）**

- [ ] 背景・目的（Why）が1行で書いてある
- [ ] 期待する成果物の形が明示されている
- [ ] やらなくてよい範囲が書かれている
- [ ] 判断が必要な箇所が明示されている
- [ ] 期限 or 優先度が明確

→ **相手の判断量を最小化**

---

## ⑤ 成果物レビュー前（Review Ready Check）
**指摘ブレ・精神消耗防止（3分）**

- [ ] 前提条件が明示されている
- [ ] 意図した判断ポイントが分かる
- [ ] 想定される質問が1つ以上潰されている
- [ ] レビューしてほしい観点が書いてある

→ レビューは**評価ではなく検証**

---

## 共通ルール（重要）
- Yes / No で終わる（考え込まない）
- 各チェックは 5項目以内
- 成果物と同じ場所に置く（探さない）
