# 思考の塊 → テストケース「自動変換の型」

- **完全に機械作業**

## テストケース変換テンプレ（固定）
```
TC-XX タイトル
Given 前提
When 操作
Then 結果
```

### 変換ルール（これを守る）
- Given / When / Then 以外を書かない
- 判断済み事項しか使わない
- 迷ったら止める

### 合言葉（変換時）
- 「これは翻訳作業」
- 「判断はもう終わっている」
- 「表にないものは書かない」
- 「迷ったら中断」

## 🧠→✋ 変換対応表（考えないで当てはめる）

| 脳内の塊・思考の種類 | 意味 / 状態 | フォーマット上の置き場所 | 具体例 |
|---|---|---|---|
| 正常 / 通る / 想定どおり | 期待する基本動作 | TC-01 正常系 | TC-01 正常 |
| NG / 弾く / エラー | 明確な異常 | 異常系TC | TC-02 異常（必須未入力） |
| 境界 / ギリギリ | 境界値 | 上限 / 下限TC | TC-03 上限超過 |
| 〜の場合 | 前提条件 | Given | Given username="" |
| 〜すると | 操作・イベント | When | When POST /users |
| 〜になる | 結果・挙動 | Then | Then 400 error |
| 不安 / 壊れそう | リスク仮説 | 異常系候補 | TC-0X 想定外入力 |
| 例外 | 特殊ケース | 例外TC | TC-0X 例外系 |
| 将来困りそう | 保守観点 | コメント or 補足 | ※将来拡張注意 |
| 判断が必要 | 未確定 | 変換しない（中断） | 要判断：〇〇 |

### 変換時ルール
- 表にない変換はしない
- 判断が出ていないものは TC にしない
- 迷ったら「要判断」と書いて作業終了
- TC は Given / When / Then の3行のみ

### 最小フォーマット
```
TC-XX タイトル
Given 前提
When 操作
Then 結果
```

### 使い方イメージ（超短縮）
- **一切考えずに変換する**
```
# 思考ダンプ
・空文字はNG
・21文字は多分ダメ
・APIでは400返したい

⬇ 対応表に当てはめるだけ ⬇

TC-02 空文字
Given username=""
When POST /users
Then 400 error

TC-03 21文字
Given username=xxxxxxxxxxxxxxxxxxxxx
When POST /users
Then 400 error
```
