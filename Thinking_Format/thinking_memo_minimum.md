# 作業切り替え・引き渡し用
- ①〜③は 作業中・中断前 に書く
- ④〜⑤は 再開時・引き渡し時 に使う
- *成果物でないので、正しさ、綺麗さは不要*
- このフォーマットの目的
  - 🧠 思考の退避
  - 🔄 作業再開の導線
  - 📄 要件定義・設計書の素材
  - 💬 レビュー時の背景説明
- *しんどい時は、「④ 現在地サマリ（作業切り替え用）」だけでもOK*
- 完成ではなく、整理が目的
  - 思考が「塊」のまま外に出せる
  - 判断点を分離できる
  - 設計書に変換するときに脳の負荷が跳ねない
  - 体調に波があっても「続きから戻れる」
- 試しに使ってみる
- 状況時応じて、「ここ要らない／足したい」を一緒に削る・足す

## 作業切り替え
### ① 今の思考ダンプ（壊れていてOK） =「広い・混沌」
- 単語だけでもOK
- 「なぜ？」を書かなくていい
- 判断途中のものはそのまま出す
- *ワーキングメモリの解放*
- 大体、以下の4点が混ざっていればOK
  - [ ] A. すでにやったこと
  - [ ] B. 今考えている途中のこと
  - [ ] C. 気になっている／詰まり
  - [ ] D. 次にやるべきこと（曖昧）

### ② 気になっている点・判断が必要な点
- モヤっと
- 違和感
- 決めきれていないこと
- *ここが後で要件・設計の「未決事項」になる*

### ③ 前提・制約（曖昧でOK）
```
・技術：
・運用：
・時間／体制：
```
- 正確でなくていい
- 「たぶん」「〜なはず」でOK

## 引き渡し
### ④ 現在地サマリ（作業切り替え用） =「再開に必要な最小情報」
- *【重要】5分で書く*
- *再開時のコンテキスト復元*
  - 作業終了 5分前アラーム
  - ミニサマリは コピペテンプレ
  - 完璧率 30%でOK

```
【作業名】
【どこまでやったか】
【今やめた理由】
【次にやる1アクション】
```

##### ミニサマリかけたかチェックリスト
- [ ] 作業名が一言で分かる
- [ ] どこまでやったかが事実ベース
- [ ] やめた理由が1行で書いてある
- [ ] 次にやることが1アクション.

#### 思考ダンプ → ミニサマリ 変換マッピング

| 思考ダンプに含まれる内容 | ミニサマリの記載欄 |
|--------------------------|--------------------|
| 作業のテーマ・対象       | 【作業名】           |
| すでにやったこと         | 【どこまでやったか】 |
| 詰まり・違和感           | 【今やめた理由】     |
| 「次は〜」「やるべきこと」| 【次にやる1アクション】 |

##### 変換時のルール
❌ やってはいけないこと
	•	きれいな文章にする
	•	正しさを確認し始める
	•	設計書レベルに落とす
	•	「結論」を出そうとする

⭕ やること
	•	そのまま拾う
	•	短く切る
	•	判断しない

### ⑤ 成果物への変換メモ（最低限）
- ここは箇条書きのみ
- 清書は別ドキュメントでやる
```
■ 要件に書くこと
- 

■ 設計で決めたこと
- 

■ 未決（レビューで聞く）
- 
```
