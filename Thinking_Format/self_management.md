# 自身の取扱説明書テンプレート

## 前提
- **「考える自分（設計者）」と「手を動かす自分（実装者）」を分離する**
  - 🧠 思考モード: 考える・決める・設計する
    - ❌　実装・調査の深掘り
  - ✋ 実行モード: 決まったことを進める
    - ❌ 新しい判断・設計
  - ⚠️ **思考と実行を同時にしない**
- 仕様を知らずに使われる → 不具合
- 仕様を共有して使う → 安定稼働

## 取扱説明書テンプレート

### 基本特性
```
・構造化・設計・整理が得意
・曖昧な指示があると判断に時間がかかる
・一貫したルールがあると安定する
```

### 得意なこと
```
・要件整理、設計レビュー
・手順書・ルール作成
・原因分析、再発防止策
・深く考える技術検討
```

### 苦手／負荷が高いこと
```
・口頭のみの指示
・前提が共有されていない議論
・急な仕様変更
・同時並行タスクが多い状態
```

### 力を発揮しやすい条件
```
・目的とゴールが明示されている
・文章や図で情報が共有される
・作業時間をブロックできる
・事前に考える時間がある
```

### 体調・疲労のサイン
```
・思考が散る
・細かいミスが増える
・音や会話がつらくなる
```

### 不調時の対応方法
```
・思考が散る
・細かいミスが増える
・音や会話がつらくなる
```

### 周囲にお願いすること
```
・指示は文章でいただけると助かります
・目的・期限・優先度を明示してください
・即答が難しい場合、後ほど整理して回答します
```

### 注意事項
```
・無理に雑談に合わせ続けない
・疲れを「気合」で乗り切らない
・調子の良さを基準にしない
```
