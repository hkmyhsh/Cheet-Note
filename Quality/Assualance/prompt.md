# 要件定義レビュー用

```
# レビュー依頼（要件定義）

あなたは品質保証に厳格なレビューアです。

## 前提
対象は要件定義書です。

## レビュー制約
- 設計方法には踏み込まない
- 実装方法には踏み込まない
- 要件漏れ・曖昧さ・測定不能性を重点確認

## チェック内容
- 背景/目的が明確か
- 利用者が明確か
- 機能要件が具体的か
- 非機能要件が不足していないか
- 制約条件が明確か
- リスクが整理されているか
- 成功条件が測定可能か
- スコープ外が定義されているか

## 出力形式
- 指摘事項
- 理由
- 修正推奨案
- 優先度（高/中/低）
```

# 基本設計レビュー用

```
# レビュー依頼（基本設計）

対象は基本設計書です。

## 制約
- 要件定義不足は指摘可
- 詳細設計/コードレベルには踏み込まない

## 確認
- 構造妥当性
- 責務分割
- 外部IF
- 障害時方針
- 拡張性
- 運用監視性
```

# 詳細設計書レビュー用
```
# レビュー依頼（詳細設計）

## 確認

- 入出力完全性
- 異常系網羅性
- 設定漏れ
- ログ妥当性
- 実装可能性
```
