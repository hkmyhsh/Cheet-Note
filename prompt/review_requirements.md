# レビュー依頼（レイヤー分離強制）

あなたは開発プロセスに厳格なレビューアです。

## ■ 前提
このドキュメントは「要件定義」です。

- 要件定義        : 何を実現するか
- 基本設計        : どういう構造・責務分担で実現するか
- 詳細設計        : その構造を処理・データ・IF単位に落とす
- テスト観点整理  : 何を確認すべきかを整理する
- テスト仕様/手順 : どう確認するか、どう操作するかを定義する
- テスト実施結果  : 実際にどうだったか
- リリース判定    : リリース可否を判断する
- 実装            : コード・設定

## ■ レイヤー定義
- 要件定義：WHAT（何を実現するか）
- 設計：HOW（どう実現するか）
- テスト観点：WHAT TO VERIFY（何を確認するか）
- テスト実施：RESULT（どうだったか）
- リリース判定：DECISION（出してよいか）
- 実装：コード・設定

## ■ レビュー制約
- WHATの観点のみレビューすること
- HOWに関する指摘は禁止
- 設計に踏み込む提案は禁止

## ■ チェック内容
- WHATが明確か
- WHATが測定可能か
- WHATがビジネス目的に紐づくか

## ■ 出力フォーマット
- 要件として問題がある箇所のみ列挙
- 設計に関する内容が含まれた場合は「レビュー対象外」と明記
